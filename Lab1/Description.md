# Лабораторная работа 1
## Условие без звездочки
Пользуясь терминалом на компьютере А перенести файл с компьютера Б на компьютер С, находящиеся в одной локальной сети при помощи ssh

## Ход работы
1. С помощью команды `ip a` определяем ip адреса каждого из компьютеров (представлено в таблице ниже).
   
| Компьютер | Имя | IP |
|---------|---|--|
| A | root |192.168.211.212|
| B | yana |192.168.211.159|
| C | kubuntu |192.168.211.166|

2. Создаем файл `HelloWorld.md` на компьютере B
![Изображение](https://github.com/Patineshka/DevOps/blob/main/Lab1/Снимки%20экрана/Снимок%20экрана%202023-11-07%20174029.png)

3. Далее с помощью команды `ssh` мы должны были подключиться к компьютерам B и C через компютер A, однако это не получилось сделать возможно по одной из следующих причин:
    1. Одна из операционных систем на флешке (Linux Tails) заточена на безопасность в сети; она доставила проблем в ходе выполнения лабы и, возможно, заблокировала менее безопасную передачу файлов с использованием паролей
    2. Вторая вероятная причина - некорректные конфигурационные файлы ssh. В ходе выполнения лабораторной работы мы несколько раз редактировали их (чтобы лучше разобраться с работой ssh и scp в частности); возможно, "зацепили" директиву, ответственную за передачу данных по паролю

В любом случае, файлы между машинами были переданы успешно (см. условие со звездочкой)
